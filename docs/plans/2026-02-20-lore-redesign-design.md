# Lore 페이지 리디자인 설계

> 작성일: 2026-02-20
> 범위: 대시보드 세계관 아카이브 페이지 UI 리디자인

## 1. 현재 상태
- `/lore` 단일 페이지, searchParams로 카테고리 전환
- 상단 칩 네비게이션 6개 + 마크다운 HTML 렌더링
- 기본 타이포그래피만, 세계관 몰입감 부족

## 2. 디자인 방향
- **터미널/데이터베이스 감성**: HELIOS 내부 아카이브 시스템
- **적당한 배합**: 헤더/네비/뱃지에만 터미널 감성, 본문은 가독성 우선
- **카드 그리드 → 모달 상세**: 카테고리별 카드 클릭 시 모달로 콘텐츠 표시

## 3. 인덱스 페이지 (`/lore`)

### 헤더
- hud-label: `HELIOS // CLASSIFIED ARCHIVE`
- 타이틀: `세계관 아카이브`
- 시스템 메시지: "접근 권한에 따라 일부 정보가 제한됩니다"

### 카드 그리드
- 데스크탑 3열 / 모바일 2열
- 각 카드: HUD 코너 브라켓, 기밀 등급 뱃지, 한글 이름, 영문 코드네임, 설명, "OPEN FILE" CTA
- hover: border glow

### 기밀 등급 (연출용)
- PUBLIC (LV.1) — 초록
- RESTRICTED (LV.2) — 노랑
- CLASSIFIED (LV.3) — 빨강

## 4. 상세 모달
- Modal 컴포넌트 활용 (createPortal)
- 헤더: 카테고리명 + 기밀 등급 뱃지 + 영문 코드네임
- 본문: lore-content 스타일 HTML (스크롤)
- 하단: 이전/다음 카테고리 네비게이션

## 5. 유연성 설계
- 카테고리 데이터를 배열 하나로 관리
- `{ id, label, codeName, description, clearanceLevel, filename }`
- 카테고리 추가/삭제/순서 변경 = 배열만 수정

## 6. 라우팅
- `/lore` 단일 유지 (변경 없음)
- searchParams 불필요 (모달 기반)
