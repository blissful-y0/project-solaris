# PROJECT SOLARIS 종합 플래닝 v2

> 작성일: 2026-02-17  
> 목적: `feat/plan-ai-model-routing` 브랜치를 "AI 라우팅 단일 주제"에서 "제품 종합 플래닝"으로 확장

## 1) 배경

기존 메뉴(`홈/전투/RP/도감/MY`)는 기능 중심이며, 신규 유저 온보딩(세계관 이해)과 러닝 커뮤니티 운영(실시간 세션/요약 브리핑) 요구를 충분히 담지 못한다.

이번 변경은 **코드 구현이 아니라 문서/설계 기준선 재정의**를 목표로 한다.

## 2) 정보구조(IA) v2

### 사용자 메인 메뉴 (IA 라벨 확정)

1. `Home` — 개인 상태 + 커뮤니티 요약 허브
2. `Lore` — 세계관 입문 허브
3. `Session` — 전투/RP 통합 허브
4. `REGISTRY` — 러닝 멤버 실시간 프로필
5. `Helios Core` — 스토리 브리핑/타임라인
6. `MY` — 내 캐릭터/설정

### Lore 페이지 위치

- `Lore`는 `/world` 경로를 사용한다.
- 신규 유저 진입 시 `Lore`를 온보딩 진입점으로 노출한다.

### 세계관 뎁스 구조

- 개요
- 사회구조
- 공명율과 능력체계
- 능력분류
- 대립구도
- 배틀룰

## 3) 핵심 기능 제안 (적합 기능)

### A. Lore 허브 (`/world`)

- 정적 문서 + 운영 공지 블록 결합
- 섹션별 앵커/진도 표시(읽은 섹션 체크)
- 신규 유저 온보딩 CTA: "캐릭터 생성으로 이동"

### B. 세션 통합 (전투/RP)

- 방 생성 시 `mode: battle | rp` 선택
- 공통 UI(방 목록/참가자/로그) + 모드별 규칙 분기
- 기존 `/battle`, `/room` 모델은 내부적으로 유지 가능하되 UX는 통합 노출

### C. REGISTRY (실시간 인물 레지스트리)

- 러닝 중 멤버 프로필 카드(접속 상태/최근 활동/소속/진행 세션)
- 상세 모달에서 전투 이력 + RP 흔적 요약
- 공개 범위(전체/진영/친구) 정책은 운영 설정으로 제어

### D. 헬리오스 코어 봇

- 최근 스토리 요약(일/주 단위)
- 사건 타임라인(핵심 전개, 전투 결과, 승인된 서사 반영)
- 운영자 코멘트 핀(수동 우선, AI 초안 보조)

### E. AI 라우팅(기존 플랜 승계)

- 기능별 고정 라우팅 유지 (`main_story`, `battle_judgment`, `lore_reflection`, `news_generation`)
- 헬리오스 코어 관련 키는 추후 `core_briefing`으로 확장 검토

## 4) 단계별 계획 (문서 기준)

### Phase A (기준선 확정)
- SERVICE/API/ADMIN 문서에서 IA v2 반영
- 전투/RP 통합 개념을 용어 수준에서 먼저 확정

### Phase B (운영 정책 고정)
- 공개 범위/권한/rate limit/알림 이벤트 정의
- 헬리오스 코어 봇의 생성-검수-발행 흐름 정의

### Phase C (구현 플랜 상세화)
- 기존 dashboard impl-plan을 IA v2 기준으로 재분해
- 작업 단위를 온보딩/세션/도감/코어로 재편성

## 5) 변경 영향

- 기존 용어 `전투`, `RP`는 사용자 IA에서 `Session`으로 통합
- 기존 `도감` 용어는 사용자 IA에서 `REGISTRY`로 전환
- `Home`의 뉴스/알림 허브 기능 일부는 `Helios Core`로 이동
- 관리자 패널에 "세계관 콘텐츠" 및 "헬리오스 코어" 운영 항목 추가 필요

## 6) 의사결정 메모

- "전부 다" 요구는 **기능 범위**로 수용하고, 구현은 리스크를 낮추기 위해 단계 분리한다.
- 본 문서는 코드 변경 전 제품 구조를 합의하기 위한 기준 문서다.
